## Prerequisites

Make sure you have the following installed before you begin:

- [Node.js](https://nodejs.org/) v20 or higher
- [npm](https://www.npmjs.com/) v9 or higher (bundled with Node.js)
- [Git](https://git-scm.com/)
- A [GitHub](https://github.com) account

Optional (only needed if working on the Cloudflare Worker):

- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/install-and-update/) — `npm install -g wrangler`
- A [Groq API key](https://console.groq.com/) for local AI summary generation

## Fork & Clone

1. Click **Fork** on the top-right of the repository page to create your own copy.

2. Clone your fork locally:

   ```bash
   git clone https://github.com/<your-username>/Rankistan.git
   cd Rankistan
   ```

3. Add the upstream remote so you can pull in future changes:

   ```bash
   git remote add upstream https://github.com/Sudo-Ali-Dev/Rankistan.git
   ```

## Local Setup

Install all dependencies from the root of the project:

```bash
npm install
```

Also install the script-specific dependencies:

```bash
cd scripts
npm install
cd ..
```

## Running the Frontend

Start the Vite development server:

```bash
npm run dev
```

The app will be available at `http://localhost:5173`. It reads leaderboard data from `public/data.json`, which is already checked in — so the UI works without running the pipeline.

To build for production:

```bash
npm run build
```

## Running the Data Pipeline

The pipeline fetches GitHub developer data and writes it to `public/data.json`. It requires a GitHub token.

1. Export your GitHub personal access token (classic or fine-grained with `read:user` scope):

   ```bash
   export GITHUB_TOKEN=your_token_here
   ```

2. Run a single incremental batch (0–23):

   ```bash
   node scripts/run-all.js --incremental 0
   ```

   Each batch covers a different cohort of developers by account creation date. Running batch `0` is enough for local testing — it does not need to be run in order.

After the script finishes, `public/data.json` will be updated and the frontend will reflect the new data on the next page load.

## Running the Cloudflare Worker Locally

The Worker powers the AI developer summary feature. You only need this if your change touches `cloudflare/worker.js`.

1. Copy the example environment file:

   ```bash
   cp cloudflare/.dev.vars.example cloudflare/.dev.vars
   ```

2. Edit `cloudflare/.dev.vars` and fill in your values:

   ```
   GROQ_API_KEY=your_groq_key_here
   SUMMARY_ALLOWED_ORIGIN=http://localhost:5173
   ```

3. Start the local Worker:

   ```bash
   cd cloudflare
   wrangler dev
   ```

4. In a separate terminal, set the frontend to point at your local Worker:

   ```bash
   VITE_SUMMARY_API_URL=http://localhost:8787 npm run dev
   ```

> **Note:** Never commit your `.dev.vars` file. It is already listed in `.gitignore`.

## Making Changes

1. Sync your fork with upstream before starting work:

   ```bash
   git fetch upstream
   git checkout main
   git merge upstream/main
   ```

2. Create a new branch for your change. Use a short, descriptive name:

   ```bash
   git checkout -b fix/leaderboard-score-display
   # or
   git checkout -b feat/add-filter-by-city
   ```

3. Make your changes. Some common areas:

   | What you want to change | Where to look                                     |
   | ----------------------- | ------------------------------------------------- |
   | UI components           | `src/components/` or `src/pages/`                 |
   | Scoring weights         | `score-config.json`                               |
   | Scoring logic           | `scripts/score.js`                                |
   | Data fetching           | `scripts/fetch-devs.js`                           |
   | AI summary API          | `cloudflare/worker.js`                            |
   | Styles                  | `src/index.css` or Tailwind classes in components |

4. Test your changes locally using `npm run dev`.

5. Commit your changes with a clear message:

   ```bash
   git add .
   git commit -m "fix: correct score display rounding on leaderboard cards"
   ```

   Commit message format: `<type>: <short description>`
   Common types: `fix`, `feat`, `chore`, `docs`, `refactor`, `style`

6. Push your branch to your fork:

   ```bash
   git push origin fix/leaderboard-score-display
   ```

## Opening a Pull Request

1. Go to your fork on GitHub and click **Compare & pull request**.

2. Set the base repository to the original Rankistan repo and the base branch to `main`.

3. Fill in the PR description:
   - What problem does this fix or what feature does it add?
   - How did you test it?
   - Screenshots or screen recordings are welcome for UI changes.

4. Submit the PR. A maintainer will review it and may leave feedback — address any requested changes by pushing new commits to the same branch.

---

## Code Style Guidelines

- The project uses **React** with functional components and hooks. Avoid class components.
- Styles are written with **Tailwind CSS** utility classes. Avoid writing custom CSS unless necessary.
- Keep components focused — if a component grows large, consider splitting it.
- Avoid committing `public/data.json` changes unless your PR is specifically about leaderboard data.
- Do not commit `.dev.vars`, `.env`, or any file containing secrets or API keys.
